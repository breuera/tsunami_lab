Additional messages can be logged during a test case. Note that the messages logged with {\ttfamily I\+N\+FO} are scoped and thus will not be reported if failure occurs in scope preceding the message declaration. An example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"Foo"}) \{}
\DoxyCodeLine{    INFO(\textcolor{stringliteral}{"Test case start"});}
\DoxyCodeLine{    \textcolor{keywordflow}{for} (\textcolor{keywordtype}{int} i = 0; i < 2; ++i) \{}
\DoxyCodeLine{        INFO(\textcolor{stringliteral}{"The number is "} << i);}
\DoxyCodeLine{        CHECK(i == 0);}
\DoxyCodeLine{    \}}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"Bar"}) \{}
\DoxyCodeLine{    INFO(\textcolor{stringliteral}{"Test case start"});}
\DoxyCodeLine{    \textcolor{keywordflow}{for} (\textcolor{keywordtype}{int} i = 0; i < 2; ++i) \{}
\DoxyCodeLine{        INFO(\textcolor{stringliteral}{"The number is "} << i);}
\DoxyCodeLine{        CHECK(i == i);}
\DoxyCodeLine{    \}}
\DoxyCodeLine{    CHECK(\textcolor{keyword}{false});}
\DoxyCodeLine{\}}
\end{DoxyCode}


When the {\ttfamily C\+H\+E\+CK} fails in the \char`\"{}\+Foo\char`\"{} test case, then two messages will be printed. 
\begin{DoxyCode}{0}
\DoxyCodeLine{Test case start}
\DoxyCodeLine{The number is 1}
\end{DoxyCode}


When the last {\ttfamily C\+H\+E\+CK} fails in the \char`\"{}\+Bar\char`\"{} test case, then only one message will be printed\+: {\ttfamily Test case start}.\hypertarget{md_submodules_Catch2_docs_logging_autotoc_md160}{}\doxysubsection{Logging without local scope}\label{md_submodules_Catch2_docs_logging_autotoc_md160}
\begin{quote}
\href{https://github.com/catchorg/Catch2/issues/1522}{\texttt{ Introduced}} in Catch 2.\+7.\+0. \end{quote}


{\ttfamily U\+N\+S\+C\+O\+P\+E\+D\+\_\+\+I\+N\+FO} is similar to {\ttfamily I\+N\+FO} with two key differences\+:


\begin{DoxyItemize}
\item Lifetime of an unscoped message is not tied to its own scope.
\item An unscoped message can be reported by the first following assertion only, regardless of the result of that assertion.
\end{DoxyItemize}

In other words, lifetime of {\ttfamily U\+N\+S\+C\+O\+P\+E\+D\+\_\+\+I\+N\+FO} is limited by the following assertion (or by the end of test case/section, whichever comes first) whereas lifetime of {\ttfamily I\+N\+FO} is limited by its own scope.

These differences make this macro useful for reporting information from helper functions or inner scopes. An example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{void} print\_some\_info() \{}
\DoxyCodeLine{    UNSCOPED\_INFO(\textcolor{stringliteral}{"Info from helper"});}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"Baz"}) \{}
\DoxyCodeLine{    print\_some\_info();}
\DoxyCodeLine{    \textcolor{keywordflow}{for} (\textcolor{keywordtype}{int} i = 0; i < 2; ++i) \{}
\DoxyCodeLine{        UNSCOPED\_INFO(\textcolor{stringliteral}{"The number is "} << i);}
\DoxyCodeLine{    \}}
\DoxyCodeLine{    CHECK(\textcolor{keyword}{false});}
\DoxyCodeLine{\}}
\DoxyCodeLine{}
\DoxyCodeLine{TEST\_CASE(\textcolor{stringliteral}{"Qux"}) \{}
\DoxyCodeLine{    INFO(\textcolor{stringliteral}{"First info"});}
\DoxyCodeLine{    UNSCOPED\_INFO(\textcolor{stringliteral}{"First unscoped info"});}
\DoxyCodeLine{    CHECK(\textcolor{keyword}{false});}
\DoxyCodeLine{}
\DoxyCodeLine{    INFO(\textcolor{stringliteral}{"Second info"});}
\DoxyCodeLine{    UNSCOPED\_INFO(\textcolor{stringliteral}{"Second unscoped info"});}
\DoxyCodeLine{    CHECK(\textcolor{keyword}{false});}
\DoxyCodeLine{\}}
\end{DoxyCode}


\char`\"{}\+Baz\char`\"{} test case prints\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{Info from helper}
\DoxyCodeLine{The number is 0}
\DoxyCodeLine{The number is 1}
\end{DoxyCode}


With \char`\"{}\+Qux\char`\"{} test case, two messages will be printed when the first {\ttfamily C\+H\+E\+CK} fails\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{First info}
\DoxyCodeLine{First unscoped info}
\end{DoxyCode}


\char`\"{}\+First unscoped info\char`\"{} message will be cleared after the first {\ttfamily C\+H\+E\+CK}, while \char`\"{}\+First info\char`\"{} message will persist until the end of the test case. Therefore, when the second {\ttfamily C\+H\+E\+CK} fails, three messages will be printed\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{First info}
\DoxyCodeLine{Second info}
\DoxyCodeLine{Second unscoped info}
\end{DoxyCode}
\hypertarget{md_submodules_Catch2_docs_logging_autotoc_md161}{}\doxysubsection{Streaming macros}\label{md_submodules_Catch2_docs_logging_autotoc_md161}
All these macros allow heterogeneous sequences of values to be streaming using the insertion operator ({\ttfamily $<$$<$}) in the same way that std\+::ostream, std\+::cout, etc support it.

E.\+g.\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{ \{c++\}}
\DoxyCodeLine{INFO( "The number is " << i );}
\end{DoxyCode}


(Note that there is no initial {\ttfamily $<$$<$} -\/ instead the insertion sequence is placed in parentheses.) These macros come in three forms\+:

{\bfseries{I\+N\+FO($\ast$$\ast$ {\itshape message expression} $\ast$$\ast$)}}

The message is logged to a buffer, but only reported with next assertions that are logged. This allows you to log contextual information in case of failures which is not shown during a successful test run (for the console reporter, without -\/s). Messages are removed from the buffer at the end of their scope, so may be used, for example, in loops.

{\itshape Note that in Catch2 2.\+x.\+x {\ttfamily I\+N\+FO} can be used without a trailing semicolon as there is a trailing semicolon inside macro. This semicolon will be removed with next major version. It is highly advised to use a trailing semicolon after {\ttfamily I\+N\+FO} macro.}

{\bfseries{U\+N\+S\+C\+O\+P\+E\+D\+\_\+\+I\+N\+FO($\ast$$\ast$ {\itshape message expression} $\ast$$\ast$)}}

\begin{quote}
\href{https://github.com/catchorg/Catch2/issues/1522}{\texttt{ Introduced}} in Catch 2.\+7.\+0. \end{quote}


Similar to {\ttfamily I\+N\+FO}, but messages are not limited to their own scope\+: They are removed from the buffer after each assertion, section or test case, whichever comes first.

{\bfseries{W\+A\+RN($\ast$$\ast$ {\itshape message expression} $\ast$$\ast$)}}

The message is always reported but does not fail the test.

{\bfseries{F\+A\+IL($\ast$$\ast$ {\itshape message expression} $\ast$$\ast$)}}

The message is reported and the test case fails.

{\bfseries{F\+A\+I\+L\+\_\+\+C\+H\+E\+CK($\ast$$\ast$ {\itshape message expression} $\ast$$\ast$)}}

AS {\ttfamily F\+A\+IL}, but does not abort the test\hypertarget{md_submodules_Catch2_docs_logging_autotoc_md162}{}\doxysubsection{Quickly capture value of variables or expressions}\label{md_submodules_Catch2_docs_logging_autotoc_md162}
{\bfseries{C\+A\+P\+T\+U\+RE($\ast$$\ast$ {\itshape expression1}, {\itshape expression2}, ... $\ast$$\ast$)}}

Sometimes you just want to log a value of variable, or expression. For convenience, we provide the {\ttfamily C\+A\+P\+T\+U\+RE} macro, that can take a variable, or an expression, and prints out that variable/expression and its value at the time of capture.

e.\+g. {\ttfamily C\+A\+P\+T\+U\+R\+E( the\+Answer );} will log message \char`\"{}the\+Answer \+:= 42\char`\"{}, while 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{int} a = 1, b = 2, c = 3;}
\DoxyCodeLine{CAPTURE( a, b, c, a + b, c > b, a == 1);}
\end{DoxyCode}


will log a total of 6 messages\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{a := 1}
\DoxyCodeLine{b := 2}
\DoxyCodeLine{c := 3}
\DoxyCodeLine{a + b := 3}
\DoxyCodeLine{c > b := true}
\DoxyCodeLine{a == 1 := true}
\end{DoxyCode}


You can also capture expressions that use commas inside parentheses (e.\+g. function calls), brackets, or braces (e.\+g. initializers). To properly capture expression that contains template parameters list (in other words, it contains commas between angle brackets), you need to enclose the expression inside parentheses\+: {\ttfamily C\+A\+P\+T\+U\+RE( (std\+::pair$<$int, int$>$\{1, 2\}) );}
